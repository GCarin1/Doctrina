# ADR 0001 — Adopt the open AGENTS.md standard as the portable substrate

- **Status:** accepted
- **Date:** 2026-06-03
- **Deciders:** project owner
- **Supersedes:** —
- **Superseded by:** —
- **Evidence:** `AGENTS.md`, `.doctrina/templates/AGENTS.md.template`, `.doctrina/templates/adapters`

## Context

Doctrina needs a portable, agent-readable file at the repository root that
encodes operational rules, commands, conventions, and boundaries. The market
in 2025–2026 produced several competing conventions for this file:

- `CLAUDE.md` (Anthropic, Claude Code).
- `.cursorrules` and `.cursor/rules/*.mdc` (Cursor).
- `.github/copilot-instructions.md` (GitHub Copilot).
- `AGENTS.md` (introduced by OpenAI in August 2025, donated to the Linux
  Foundation's Agentic AI Foundation, and announced in the LF press release
  of 9 December 2025 as adopted by more than 60,000 open-source projects and
  agent frameworks including Amp, Codex, Cursor, Devin, Factory, Gemini CLI,
  GitHub Copilot, Jules, and VS Code; platinum members include AWS,
  Anthropic, Google, Microsoft, and OpenAI).

Inventing a proprietary `.doctrina/agents.md` or similar would give Doctrina
total control over the format but would cost the project free portability
across all major coding agents and IDEs.

## Decision

Doctrina adopts the open AGENTS.md standard as the primary, agent-readable
operational source of truth. The root `AGENTS.md` is mandatory for any
Doctrina-compliant repository. Per-agent adapter files (`CLAUDE.md`,
`.cursor/rules/*.mdc`, etc.) may be generated by the CLI as thin wrappers
that include or symlink the canonical AGENTS.md.

## Alternatives considered

1. **Proprietary `.doctrina/rules.md`.** Rejected. Loses portability with
   no compensating gain.
2. **Multiple peer files (CLAUDE.md, AGENTS.md, .cursorrules) all hand-edited.**
   Rejected. Guarantees drift between equivalents and creates implicit
   ownership conflicts.
3. **AGENTS.md as canonical + generated adapters for each agent.** Accepted.

## Consequences

**Positive**

- Free interoperability with Claude Code, OpenAI Codex CLI, Cursor, Copilot,
  Gemini CLI, Devin, and many others on day one.
- Doctrina inherits the AGENTS.md ecosystem of editor support and conventions.
- Reduces the surface area Doctrina has to document and maintain.

**Negative**

- Doctrina cannot evolve the format unilaterally; format changes must be
  proposed upstream.
- Some agents (e.g. Claude Code) prefer `CLAUDE.md`; users get the best
  experience only when Doctrina generates an adapter that points at AGENTS.md.

**Neutral**

- The 150-line soft limit on AGENTS.md (Cursor token-tax guidance) becomes a
  Doctrina convention.

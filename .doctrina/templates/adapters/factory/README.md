# Factory AI adapter

Factory AI reads `AGENTS.md` natively as part of its support
for the open AGENTS.md standard (per the Linux Foundation's
December 2025 announcement). No adapter file is installed for
Factory.

This directory exists so the templates inventory is symmetric
across supported agents, and so future Factory-specific
affordances have an obvious home.

If you are reading this because `doctrina init --agent factory`
did not create any files: that is the intended behaviour.

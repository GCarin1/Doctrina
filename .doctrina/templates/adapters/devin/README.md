# Cognition Devin adapter

Cognition Devin reads `AGENTS.md` natively as part of its
support for the open AGENTS.md standard (per the Linux
Foundation's December 2025 announcement). No adapter file is
installed for Devin.

This directory exists so the templates inventory is symmetric
across supported agents, and so future Devin-specific
affordances (custom session configs, for example) have an
obvious home.

If you are reading this because `doctrina init --agent devin`
did not create any files: that is the intended behaviour.

# Codex CLI adapter

OpenAI Codex CLI reads `AGENTS.md` natively (Codex is one of the
reference implementations of the open AGENTS.md standard). No adapter
file is installed for Codex.

This directory exists so the templates inventory is symmetric across
supported agents, and so future Codex-specific affordances (for example
custom subagent definitions) have an obvious home.

If you are reading this because `doctrina init --agent codex` did not
create any files: that is the intended behaviour.

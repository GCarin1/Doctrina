# Google Jules adapter

Google Jules (autonomous coding agent) reads `AGENTS.md`
natively as part of its support for the open AGENTS.md standard
(per the Linux Foundation's December 2025 announcement). No
adapter file is installed for Jules.

This directory exists so the templates inventory is symmetric
across supported agents, and so future Jules-specific
affordances have an obvious home.

If you are reading this because `doctrina init --agent jules`
did not create any files: that is the intended behaviour.
